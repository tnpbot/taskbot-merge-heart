Restoring a Backup

How to replace a changed _styles.js with your backup file.

What is a backup?

When you click ⬇ Backup in the Style Config, it downloads a copy of your current _styles.js named _styles.backup.js. This is a plain JavaScript file — no special tools needed to restore it.

How to restore

  1. 1
    Locate your backup file
    Find _styles.backup.js in your Downloads folder (or wherever your browser saves files).
  2. 2
    Open your taskbot project folder
    Navigate to the folder that contains _styles.js — the same folder as this config.html file.
  3. 3
    Rename the backup file
    Rename _styles.backup.js to _styles.js. You can optionally rename the current (broken) _styles.js to _styles.old.js first, just to keep it around.
  4. 4
    Confirm the replacement
    If your OS asks whether to replace the existing file, confirm yes. Your project folder should now contain the restored _styles.js.
  5. 5
    Refresh your OBS browser source
    In OBS, right-click your browser source → Refresh. The restored styles will load immediately.

Keeping backups organized

Each backup download is always named _styles.backup.js. If you want to keep multiple backups (e.g. before each stream), rename the file after downloading:

  1. 1
    Download the backup as usual
    Click ⬇ Backup in the Style Config before making changes.
  2. 2
    Rename with a date or label
    For example: _styles.backup.2025-10-14.js or _styles.backup.forest-theme.js. Store these in a backups/ subfolder inside your project if you like — OBS won't read them.
Tip: The Style Config only edits the values inside _styles.js — it never touches any other file. If something looks wrong with your overlay after saving, restoring _styles.js is all you need to do.
Note: The backup contains your styles at the moment you clicked ⬇ Backup. Any saves made after that point are not included — download a new backup whenever you reach a state you're happy with.
